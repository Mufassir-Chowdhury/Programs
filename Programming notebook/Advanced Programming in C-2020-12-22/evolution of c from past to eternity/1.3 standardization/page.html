STANDERDS:

There are many standards for C language.

1. GCC

It's a compiler collection and a toolchain. the GCC is a foundation that can be used to implement any compiler of any language for any hardware platform and any operating system and it can be used to implement a cross compiler. It'a free and was created by Richard Stallman in May 23. 1987. It was first called GNU C Compiler and then changed into GNU Compiler Collection.

2. CLANG

It's part of the LLVM (Low Level Virtual Machine) project. It's free too and has tested to be faster than GCC.

3. ICC

ICC is short for Intel C Compiler and is highly optimized for Intel processors. Intel claims that it's binary code is 50% faster than that generated by GCC or MS compilers.

4. XL C/C++ compiler

It's also a specialized and optimized compiler but for IBM hardware.

5. Microsoft C/C++ compiler

Created in 1983 . Later it changed it's name to CL.

6. Watcom C compiler

Created by watcom International Corporation in 1988. It was closed at first then it became free. It's still under continuous development.


It is possible that a program written for a compiler works differently or doesn't work at all in other compiler.

There are two international bodies who control the development of C language.

1. ANSI

American National Standards Institute established in 1918 with headquarters in Washington, DC.

2. ISO

International Organization for Standardization established in 1947 with headquarters in Geneve, Switzerland. It is present in 162 countries. And the ISO is not an abbreviation but rather just a name.


At the beginning, the language had no standards and no official documents. The standard was served by a book authored by Brian W. Kernighan and Dennis Ritchie, titled the C Programming Language and published for the first time in 1978. This version was called K&R "C"
The first document was published by ANSI in 1989 and was titled ANSI X3.159-1989 "Pogramming Language C". It was reffered to ANSI C and also C89.


The computers can be thought of 2 types. They differ in the way in which multibyte data is arranged in the memory.
- Machines using the lilttle endian places the lower byte containing the less significant bits at the lesser address. And Machines using the big endian containing the more significant bits at the geater address. For example, the number 1 in the little endian will be stored as,

0x01000000

instead of

0x00000001

In 1990, ISO adopted the ANSI C standard and after some changes, published it as ISO/IEC 9899:1990 which was later called C90. The C89 and C90 was in fact the same standard.

In 1995, ISO published an extended emendment to the C90 standard released as ISO/IEC 9899/AMD1:1995 and was adopted under the code name C95. It included-
  • Wide characters
  • Digraphs

The C99 standard standard was introduced in 2000 by the document ISO/IEC 9899:1999. It included-
  • New data types
  • Static array indices, designated initializers, compound literals, variable-length arrays, flexible array members, variadic macros and the restrict keyword
  • C++ styled comments
  • for loops with internal control variable declaration.

The C11 is the last C standard by the document ISO/IEC 9899:2011. It included-
  • Unicode support
  • the _Generic keyword
  • Platform independent multi-threading tools
  • Generic types


How to know the version you are using:
  • The C89/C90 defines a symbol __STDC__
  • The value of __STDC_VERSION__ for C95 should be 199409L
  • The value of __STDC_VERSION__ for C99 should be 199901L
  • The value of __STDC_VERSION__ for C11 should be 201112L
