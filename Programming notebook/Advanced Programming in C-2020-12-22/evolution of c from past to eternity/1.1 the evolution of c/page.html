EVOLUTION:

Few language concepts were inconvenient. For example, data initializers looked like

int count(1);

instead of

int count = 1;

which was identical to function invocation. Function invocation was different too. Like,

int function(n) int n;{}

instead of

int function(int n){}

though both practice are accepted. But, for the older one, the compiler doesn't check if the numbers and types of the argument is right or wrong. The following three invocations will be compiles.
  • function();
  • function(1);
  • function(1, 2);

The extra arguments will be ignored and the missing ones will have garbage values.

That's why one should use void instead of empty paranthesis. Because, the compiler will consider it to be written in old style instead.
