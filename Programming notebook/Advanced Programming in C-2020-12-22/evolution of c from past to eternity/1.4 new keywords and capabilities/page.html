_Noreturn keyword

This keyword is a function specifier. It tells the compiler that the function will not return to its invoker, so the compiler gets ready for the consequences. For example, exit()function doesn't return to the invoker.

CL 19.x doesn't support this keyword

_Bool keyword

This introduces a new data type to store only 0 and 1 representing true and false. This uses a single bit. Used in expressions, this type is treated as unsigned ints. Any nonzero values when converted to a _Bool type is treated as 0.
N.B: There are no additional keywords for representing literals for true or false.

_Exit function

This function is a more powerful function than exit()though it stills needs te header stdlib.hto be included. Another function named atexit()calls the provided function(s) when exit() is called. But, _Exit()forces the program to tarminate without calling functions using atexit()

_Generic keyword

This was added in C11 and named generic selection. It decides which entity should be chosen depending on the expression's type. The form is,
_Generic(selector, type1: item1, type2: item2, ... , default: itemn)
The default branch is optional.

CL 19.x doesn't support this keyword

__func__ symbol

C11 added a new symbol __func__ alongside others like __FILE__ and __LINE__. This symbol represents a string containing the name of the enclosing function.

_Complex keyword

This keyword refers to the complex numbers from mathematics. It is a synonym of double _Complex but can be used with conjuction of float or long double. The imaginary part contains an i suffix. The mathematical operators works fine with complex numbers and few other complex fariants is added to some commonly used functions. I.e,: cabs(), cexp(), sqrt()and so on. Complex numbers are easier to use with complex.h header.

CL 19.x doesn't support this keyword
Neither scanf() nor printf()have specific support for inputting/ouputting complex.
